The "Old Way" - Life Before React

Before React, we built websites using Vanilla JavaScript or libraries like jQuery.

  • We directly told the browser what to change: "Find this element, change its text."
  • This is called direct DOM manipulation.
  • It's like giving step-by-step instructions to rebuild a wall, brick by brick.
  • This worked for small sites, but became a nightmare for complex applications.

Why the "Old Way" Was Hard

State Management Chaos: Keeping track of data and ensuring the UI matched it was incredibly difficult.

"Spaghetti Code": Code became a tangled mess, making it hard to debug or add new features.

Slow Performance: Every small change could force the browser to recalculate and redraw large parts of the page.

UI and Data Out of Sync: It was easy for the data to change but the UI to not update, causing countless bugs.

üí° The Goal: A new way to build UIs that were predictable, efficient, and easier to manage.

React's Big Idea: The Virtual DOM

Instead of touching the real DOM for every change, React introduced a brilliant concept.

  • The Virtual DOM (VDOM) is a lightweight copy of the real browser DOM.
  • It's just a JavaScript object representing what the UI should look like.
  • Because it's an object, reading from it and writing to it is incredibly fast.

üß† Analogy: The real DOM is the final printed book. The Virtual DOM is the editable draft in a word processor. It's much faster to change the draft.

How the Virtual DOM Works: The "Diffing" Process

When your app's data (state) changes, React performs a simple, super-fast process:

1. State Changes
‚Üí
2. New Virtual DOM is created
‚Üí
3. "Diffing" compares new vs. old VDOM
‚Üí
4. Only changes are sent to Real DOM

This process is why React is so fast. It minimizes the slow and expensive work of touching the real browser DOM.

One Page, Infinite Views: The Single Page App (SPA)

Traditional Website

  • Loads a new HTML page from the server for every link click.
  • Causes a full-page refresh (white screen flash).

React SPA

  • Server sends just one HTML file.
  • Navigation just swaps out components on the screen.
  • No page reloads!

The result? A super smooth, fast, app-like experience for the user.

Passing Data with "Props"

Components need data. That's where props (properties) come in.

  • Props pass data from a parent component down to a child component.
  • Props are read-only. A child cannot change the props it receives.
// Parent component
    function App() {
      return <ProfileCard username="Tapabrata" />;
    }

    // Child component receives 'username' as a prop
    function ProfileCard(props) {
      return <h2>Welcome, {props.username}!</h2>;
    }

Meet JSX: HTML + JavaScript

JSX (JavaScript XML) lets you write HTML-like code directly inside your JavaScript.

Without JSX (Harder)

return React.createElement(
      'h1',
      null,
      'Hello, Tapabrata!'
    );

With JSX (Easier)

const name = "Tapabrata";
    return <h1>Hello, {name}</h1>;

üß† Benefit: JSX makes your code more readable and lets you visualize the UI structure while you're coding.

A Component's Memory: "State"

If props are data given to a component, state is the data that a component manages itself.

  • State is a component's private, internal memory.
  • When the state of a component changes, React automatically re-renders that component to show the new data.
  • This is the key to creating dynamic, interactive UIs!

Example: A counter component needs to remember the current count. That count is its state.

Navigation with React Router

How do we handle different "pages" (like `/home`, `/about`) in a Single Page App? With React Router.

  • It syncs your UI with the URL in the browser's address bar.
  • When the URL changes, React Router swaps out the component being displayed without a full page reload.
import { BrowserRouter, Routes, Route } from 'react-router-dom';

    function App() {
      return (
        <BrowserRouter>
          <Routes>
            <Route path="/" element={<HomePage />} />
            <Route path="/about" element={<AboutPage />} />
          </Routes>
        </BrowserRouter>
      );
    }

Function Component Superpowers: Hooks

In the past, to use state and other features, you had to use JavaScript class components.

Hooks are special functions that let you "hook into" React features right from your functional components.

  • They make your code cleaner, shorter, and easier to understand.
  • All hooks start with the word use (e.g., `useState`, `useEffect`).

Core Hook #1: useState

The most important hook. It lets you add state (memory) to your functional components.

useState returns an array with two things: the current state value, and a function to update it.

import { useState } from 'react';

    function Counter() {
      // `count` holds the value (starts at 0)
      // `setCount` is the function to change it
      const [count, setCount] = useState(0);

      return (
        <div>
          <p>You clicked {count} times</p>
          <button onClick={() => setCount(count + 1)}>
            Click me
          </button>
        </div>
      );
    }

Core Hook #2: useEffect

What if you need to do something *after* React renders? Like fetching data, setting subscriptions, or manually changing the DOM.

That's a "side effect," and `useEffect` is the hook for it.

import { useState, useEffect } from 'react';

    function UserProfile() {
      const [user, setUser] = useState(null);

      useEffect(() => {
        // This code runs after render
        fetch('api.example.com/user/1')
          .then(res => res.json())
          .then(data => setUser(data));
      }, []); // [] means run only once

      if (!user) return <p>Loading...</p>;
      return <h1>Hello, {user.name}</h1>;
    }

Core Hook #3: useRef

useRef is a box for a value that persists across renders, but changing it does not trigger a re-render.

Use Case 1: Accessing DOM elements directly (e.g., to focus an input).
Use Case 2: Storing a mutable value without re-rendering (e.g., a timer ID).
import { useRef, useEffect } from 'react';

    function MyInput() {
      const inputRef = useRef(null);

      useEffect(() => {
        // Focus the input element after mount
        inputRef.current.focus();
      }, []);

      return <input ref={inputRef} type="text" />;
    }

Next Steps: Global State with the Context API

Problem: Passing props through many layers of components ("prop drilling") is painful.

Solution: Context creates a global state that any component can access without needing props.

You create a Provider at the top level to hold the value. Any child can then use the useContext hook to read it.

Perfect for: Theme (dark/light mode), user authentication, language preference.

Next Steps: Reusing Logic with Custom Hooks

Problem: You find yourself writing the same stateful logic over and over in different components.

Solution: Extract that logic into your own reusable hook!

// The custom hook (starts with 'use')
    function useFetch(url) {
      const [data, setData] = useState(null);
      useEffect(() => { /* ... fetch logic ... */ }, [url]);
      return data;
    }

    // How to use it in any component
    function MyComponent() {
      const users = useFetch('/api/users');
      // ... use the 'users' data
    }

Next Steps: Performance with Memoization

Problem: Some components might re-render too often, slowing things down.

Solution: Memoization is a technique to skip re-rendering if props haven't changed.

React.memo(): A wrapper for your component that prevents re-rendering if props are the same.

useMemo(): A hook to memoize a calculated value so it's not re-computed on every render.

useCallback(): A hook to memoize a function so it's not re-created on every render.

üí° Goal: Render only what's needed, when it's needed.

Next Steps: Faster Load Times with Lazy Loading

Problem: Downloading all your JavaScript code at once can be slow for large apps.

Solution: Code Splitting lets you split code into smaller chunks that are loaded on demand.

import React, { Suspense, lazy } from 'react';

    const AboutPage = lazy(() => import('./AboutPage'));

    function App() {
      return (
        <div>
          <Suspense fallback={<p>Loading...</p>}>
            <AboutPage />
          </Suspense>
        </div>
      );
    }
